怎么更好的准备面试

投递简历

准备面试之前，我们需要先确定自己期望的职位。然后需要去投递自己的简历。我会先简单介绍一下怎么投简历。

公司比较有效的招人渠道主要有5个分别是：

  • 熟人推荐
  • 垂直社交媒体
  • 传统网络招聘（Boss直聘、拉勾，牛客网）
  • 猎头（中高端）
  • 校园招聘会

其中熟人推荐成功率最高，你们可以通过找学长，直接向他所在的公司进行内推，而且每当到了招聘季，大家都会发布内推码之类的信息。想到字节的可以找我。可以收藏下这个内推码或者这个链接：

面试综述

简历筛选通过之后，就可以准备面试了。

一个典型的校招面试，通常包含两轮专业能力面试和一轮综合面试（专业能力+软素质）。每一轮的面试时间大概在40分钟左右。面试的内容主要包括：

  • 基础知识考查
  • 算法&数据结构+编码
  • 大学期间的研究内容或做过的项目考察
  • 职能专项

面试是一个相互选择的过程。对面试官和候选人的尊重是最基本的原则。要做好换位思考，思考“如果我是面试官”，应该怎么样。所以对于面试，最好做到穿着整洁，行为得体，保持一个良好的精神面貌。然后要对自己的简历做到烂熟于心，找一个安静的地方，进行面试。

基础知识

基础知识的考查主要有工程场景中常用的：操作系统，计算机网络，数据库几大类。

分级有三级：入门、基础和进阶。入门题就是这个方向基本的知识和概念。基础题涵盖了这个方向常见的知识点，是实际工作中常用到的。进阶就是偏深和原理性的点，校招的要求是入门内容大部分都了解，基础知识要有部分掌握。

在考察的时候，面试官会根据你简历所写的内容选择合理的考查方向和内容，内容不会很单一，可能会涉及很多不同的知识点，我们实际在回答的时候可以灵活回答。对于面试问题，如果遇到了自己了解或掌握的那肯定最好了。如果没有遇到，是我们不会的问题，那就承认自己不会，聊一聊自己深入掌握的方向。毕竟对于面试官来说，他们更多的是为了考察处候选人懂什么，而不是不懂什么。

  • 操作系统
    • 字符
    • 进程和线程相关
    • 内存管理
  • 网络
    • 网络基础
    • 网络层
    • 传输层
    • 应用层
    • DNS
  • 数据库
    • 简单查询，基本概念
    • 范式定义
    • 使用过的数据库详细的原理或者使用方法

编码与语言

编码和语言的考察，主要是看我们在代码设计、开发效率和代码质量方面的能力，能不能熟练使用编程语言完成开发任务以及对语言底层原理的理解。是属于必考的部分，一般会结合算法或者设计题目一起考察。主要看我们能不能把思路和逻辑用代码清晰的描述出来。

编码能力：

  • 逻辑正确
  • 基本无语法错误
  • 结构清晰可阅读
  • 抽象设计，风格良好
  • 能考虑到大部分异常边界
  • 可扩展性强

常用语言：

  • C/C++
  • JAVA
  • Golang
  • Python
  • Objective-C
  • JavaScript

精通其中的一个。

算法和数据结构

算法和数据结构是校招考察很重要的一个模块。原则上，这方面的考察时不会超出我们大学课程的范围的，除非有相关的背景。所以需要熟悉常见的基础算法和数据结构。难度大概是LeetCode中等难度。

考察点：

  • 时间复杂度/空间复杂度分析与计算
  • 常用排序
  • 基础数据结构
    • 链表
    • 队列
    • 堆
    • 栈
    • 哈希表
    • 字典树
    • 二叉树
    • 位图
  • 进阶算法
    • 动态规划
    • 布隆过滤
  • 进阶数据结构
    • B树
    • 红黑树

项目

如果你的简历中有项目经验，面试的时候还会问到你曾经做过的项目。你需要能够清楚的介绍做过的项目，自己做了哪一部分，项目中用到了哪些知识，项目中存在的不足、难点或优化点有哪些，有没有更优的方式。

软素质

除了专业能力的考察，面试时候也会对我们的软实力，软素质进行考察。概括来说就是那些暂时没办法难以估量的能力。

例如一下的几点：

  • 文化契合度
  • 沟通能力
  • 团队协作能力
  • 结构化思维能力
